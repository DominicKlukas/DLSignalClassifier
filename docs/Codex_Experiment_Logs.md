## Experiment 1: Time CNN vs FFT CNN on Time-Feature-Rich vs Frequency-Feature-Rich Signals

### Goal

Test the claim that:

- a time-domain CNN should perform better on signals whose class structure is primarily in time-domain behavior, and
- an FFT-domain CNN should perform better on signals whose class structure is primarily in frequency-domain structure.

### Constraint

- `old_experiments` was not used for this run.
- The experiment was rebuilt from the active source files:
  - [SignalGenerator.py](/home/klukasdh/Projects/DLSignalClassifier/SignalGenerator.py)
  - [WaveformFamilyGenerator.py](/home/klukasdh/Projects/DLSignalClassifier/WaveformFamilyGenerator.py)

### Datasets Used

Two fresh balanced datasets were generated by [run_experiment1_time_vs_fft.py](/home/klukasdh/Projects/DLSignalClassifier/experiments/run_experiment1_time_vs_fft.py):

1. Time-feature-rich modulation dataset
   Classes:
   - `BPSK`
   - `QPSK`
   - `8PSK`
   - `16QAM`
   - `64QAM`
   - `PAM4`

2. Frequency-feature-rich waveform-family dataset
   Classes:
   - `CW`
   - `AM`
   - `FM`
   - `OFDM`
   - `LFM_CHIRP`
   - `DSSS`
   - `FHSS`
   - `SC_BURST`

Dataset artifacts:

- [modulation_time_features.h5](/home/klukasdh/Projects/DLSignalClassifier/experiments/experiment1/modulation_time_features.h5)
- [waveform_frequency_features.h5](/home/klukasdh/Projects/DLSignalClassifier/experiments/experiment1/waveform_frequency_features.h5)

### Model Setup

- Same 1D CNN backbone used for both representations.
- Input representation changed between:
  - raw IQ time series
  - real/imag FFT
- `500` examples per class
- `40` epochs
- batch size `128`
- AdamW with cosine LR schedule

### Results

#### Modulation dataset

- Time CNN
  - best epoch: `37`
  - best val acc: `0.767`
  - test acc: `0.730`
  - macro F1: `0.727`
  - weighted F1: `0.727`

- FFT CNN
  - best epoch: `3`
  - best val acc: `0.187`
  - test acc: `0.193`
  - macro F1: `0.149`
  - weighted F1: `0.149`

#### Waveform-family dataset

- Time CNN
  - best epoch: `34`
  - best val acc: `0.580`
  - test acc: `0.598`
  - macro F1: `0.584`
  - weighted F1: `0.584`

- FFT CNN
  - best epoch: `30`
  - best val acc: `0.728`
  - test acc: `0.728`
  - macro F1: `0.728`
  - weighted F1: `0.728`

### Interpretation

This experiment supports the intended directional claim:

- On the time-feature-rich modulation dataset, the time CNN was dramatically better than the FFT CNN.
- On the frequency-feature-rich waveform-family dataset, the FFT CNN outperformed the time CNN.

The longer run made that separation clearer than the earlier short pilot:

- modulation gap: `0.730` vs `0.193`
- waveform-family gap: `0.728` vs `0.598`

So the representation mismatch shows up clearly:

- time-domain CNNs are strong when class information is tied to symbol timing / temporal evolution,
- FFT-domain CNNs are stronger when class information is tied more to spectral shape / occupancy / frequency structure.

## Experiment 2: Multimodal IQ + FFT CNN

### Goal

Test whether a fused model that processes both:

- raw IQ time series, and
- FFT real/imag features

can outperform the single-representation CNNs from Experiment 1 on both datasets.

### Architecture

Implemented in [run_experiment2_multimodal.py](/home/klukasdh/Projects/DLSignalClassifier/experiments/run_experiment2_multimodal.py).

Structure:

- one CNN branch for IQ input,
- one CNN branch for FFT input,
- each branch produces a `256`-dimensional feature vector,
- the two feature vectors are concatenated,
- a fusion MLP performs final classification.

The datasets and train/val/test splits were exactly the same as Experiment 1.

### Results

#### Modulation dataset

- Multimodal IQ+FFT CNN
  - best epoch: `18`
  - best val acc: `0.713`
  - test acc: `0.707`
  - macro F1: `0.702`
  - weighted F1: `0.702`

Comparison to Experiment 1:

- Time CNN: test acc `0.730`
- FFT CNN: test acc `0.193`
- Multimodal CNN: test acc `0.707`

#### Waveform-family dataset

- Multimodal IQ+FFT CNN
  - best epoch: `30`
  - best val acc: `0.820`
  - test acc: `0.800`
  - macro F1: `0.800`
  - weighted F1: `0.800`

Comparison to Experiment 1:

- Time CNN: test acc `0.598`
- FFT CNN: test acc `0.728`
- Multimodal CNN: test acc `0.800`

### Interpretation

The outcome is mixed, but informative:

- On the time-feature-rich modulation dataset, the multimodal model did **not** beat the time CNN.
- On the frequency-feature-rich waveform-family dataset, the multimodal model beat **both** single-mode baselines.

So the multimodal fusion did not behave like a universal free improvement. Instead:

- when one representation already dominates strongly, adding the weaker representation may not help and can slightly hurt,
- when both representations contain useful but different evidence, multimodal fusion can produce a real gain.

### Conclusion

Experiment 2 supports a narrower claim than the original hypothesis:

- the multimodal IQ+FFT CNN is **not** automatically better on every dataset,
- but it is clearly effective on the waveform-family task, where both time and frequency structure appear to matter,
- and it underperforms the pure time CNN on the modulation task, where the FFT branch mostly adds noise rather than complementary information.

## Experiment 3: Gated Multimodal IQ + FFT CNN

### Goal

Test whether a safer multimodal architecture can:

- stay close to the best single-mode model when one branch is mostly unhelpful,
- still use both inputs when fusion is genuinely useful.

### Architecture

Implemented in [run_experiment3_gated_multimodal.py](/home/klukasdh/Projects/DLSignalClassifier/experiments/run_experiment3_gated_multimodal.py).

Structure:

- one IQ feature branch,
- one FFT feature branch,
- an IQ-only classifier head,
- an FFT-only classifier head,
- a fused classifier head,
- a learned gate over the three logit sources,
- mild branch dropout during training,
- auxiliary losses on the branch heads and fusion head.

Final prediction:

- weighted combination of IQ logits, FFT logits, and fused logits.

### Results

#### Modulation dataset

- Gated multimodal CNN
  - best epoch: `11`
  - best val acc: `0.670`
  - test acc: `0.680`
  - macro F1: `0.676`
  - weighted F1: `0.676`

Comparison:

- Time CNN: `0.730`
- Plain multimodal CNN: `0.707`
- Gated multimodal CNN: `0.680`

Gate behavior:

- best-val gate weights: `[0.998, 0.001, 0.001]`
- test gate weights: `[0.998, 0.001, 0.001]`

Interpretation:

- the model correctly learned that modulation should be almost entirely IQ-driven,
- but the gated/auxiliary formulation still did not recover the pure time-CNN baseline.

#### Waveform-family dataset

- Gated multimodal CNN
  - best epoch: `30`
  - best val acc: `0.898`
  - test acc: `0.908`
  - macro F1: `0.908`
  - weighted F1: `0.908`

Comparison:

- Time CNN: `0.598`
- FFT CNN: `0.728`
- Plain multimodal CNN: `0.800`
- Gated multimodal CNN: `0.908`

Gate behavior:

- best-val gate weights: `[0.117, 0.000, 0.882]`
- test gate weights: `[0.117, 0.000, 0.883]`

Interpretation:

- the model relied primarily on the fused path,
- with a smaller but nontrivial IQ contribution,
- and almost no direct FFT-only logit contribution.

### Conclusion

Experiment 3 improved the multimodal architecture substantially, but not in the fully symmetric way we wanted:

- it still failed to beat the pure time CNN on the modulation dataset,
- but it strongly beat every previous model on the waveform-family dataset.

So the architecture is a clear win when cross-representation fusion is truly useful, but it is still not a guarantee of matching the strongest single-mode expert on every task.

## Experiment 4: Spectrogram CNN

### Goal

Test a standard spectrogram representation on the same two tasks:

- time-feature-rich modulation classification,
- frequency-feature-rich waveform-family classification.

This checks whether a time-frequency image representation is a better compromise than pure IQ or pure FFT.

### Architecture

Implemented in [run_experiment4_spectrogram.py](/home/klukasdh/Projects/DLSignalClassifier/experiments/run_experiment4_spectrogram.py).

Representation:

- complex IQ signal converted to log-magnitude STFT spectrogram
- `n_fft = 128`
- `hop_length = 32`

Model:

- small 2D CNN over the spectrogram
- global average pooling
- MLP classifier

### Results

#### Modulation dataset

- Spectrogram CNN
  - best epoch: `35`
  - best val acc: `0.317`
  - test acc: `0.280`
  - macro F1: `0.262`
  - weighted F1: `0.262`

Comparison:

- Time CNN: `0.730`
- FFT CNN: `0.193`
- Spectrogram CNN: `0.280`

Interpretation:

- spectrograms are still a poor representation for this modulation task,
- but they are slightly better than pure FFT here,
- and still far worse than raw IQ.

#### Waveform-family dataset

- Spectrogram CNN
  - best epoch: `38`
  - best val acc: `0.805`
  - test acc: `0.780`
  - macro F1: `0.781`
  - weighted F1: `0.781`

Comparison:

- Time CNN: `0.598`
- FFT CNN: `0.728`
- Plain multimodal CNN: `0.800`
- Gated multimodal CNN: `0.908`
- Spectrogram CNN: `0.780`

Interpretation:

- spectrograms work reasonably well on the waveform-family task,
- they outperform both single-mode time CNN and FFT CNN,
- but they still do not match the best multimodal fusion model.

### Conclusion

Experiment 4 shows that spectrograms are a middle-ground representation:

- they are clearly not good enough for the modulation task compared with raw IQ,
- they are quite effective for the waveform-family task,
- but in this codebase they are still weaker than the best gated multimodal model.

So the ranking now looks like:

- modulation: `Time CNN > Gated multimodal > Plain multimodal > Spectrogram > FFT`
- waveform-family: `Gated multimodal > Plain multimodal > Spectrogram > FFT > Time`

## Experiment 5: Real Sub-GHz Dataset Benchmark

### Goal

Compare the three main representation families on the new real captured dataset:

- IQ time-domain CNN,
- FFT CNN,
- gated IQ+FFT multimodal CNN.

This tests whether the ranking from the synthetic experiments carries over to real Sub-GHz recordings.

### Dataset Setup

Resulting window counts:

- Gated multimodal CNN
  - best epoch: `16`
  - best val acc: `0.793`

  - weighted F1: `0.796`
  - best-val gate weights: `[0.353, 0.545, 0.101]`

### Interpretation

The real dataset partially matches the synthetic story, but with a narrower gap:

- the FFT CNN is still clearly the weakest of the three,
- the IQ CNN is strong on the real captures,
- the gated multimodal model only barely beats the IQ CNN on test accuracy.

So unlike the waveform-family synthetic benchmark, multimodal fusion is **not** a dramatic win here. The gain over raw IQ is only marginal:

- Time CNN: `0.814`
- Gated multimodal CNN: `0.816`

The macro F1 scores are much lower than the overall accuracies, which suggests the dataset remains class-imbalanced and the minority classes are substantially harder than the dominant `80211ah` and `sunOFDM` groups.

One interesting detail is that the learned gate still placed most weight on the FFT branch, even though standalone FFT underperformed badly. That suggests the FFT representation contributes complementary cues in fusion, but not enough to become a strong single-mode classifier by itself.

### Conclusion

On this real captured benchmark, the ranking is:

- `Gated multimodal ~= Time CNN > FFT CNN`

So the real-data result is more conservative than the best synthetic waveform-family result:

- multimodal fusion still helps a little,
- but the improvement over raw IQ is small,
- and raw IQ remains the most reliable single representation.

- the gated model stayed best over most of the usable SNR range,
- but its advantage did not widen nearly as dramatically at the lowest SNR values,
- and all models eventually collapsed toward chance.

The likely reason is that the real dataset contains stronger modality complementarity under stress:

- real captured protocols include hardware effects, channel effects, burst timing structure, and protocol-specific quirks,

- which gives the gated multimodal model more opportunity to adaptively rely on the stronger representation.

The synthetic waveform-family dataset is more homogeneous:

- classes are more idealized,
- label structure is already strongly frequency-shaped,
- and the nuisance variation is generator-controlled rather than measurement-driven.

So on the synthetic benchmark, multimodal fusion still helps, but severe AWGN tends to erase class evidence for all models in a more uniform way.

This gives a useful combined interpretation:

- Experiment 7 suggests multimodal fusion is especially valuable for noisy real-world RF classification,
- while Experiment 8 suggests that on idealized synthetic waveform families, fusion improves accuracy but does not create the same large low-SNR robustness margin.
